## 1. 질문 게시판 공지 로직 흐름 분석

질문 게시판에서 공지를 가져와 뿌리는 흐름은 아래 4단계로 구성됨.

```
[admin-mapper.xml] selectTopNotice
    → NOTICE 테이블에서 STATUS='Y'인 전체 공지를 NOTICE_NO DESC 정렬로 조회

[AdminMapper.java] selectTopNotice()
    → XML 쿼리에 매핑된 인터페이스 메서드

[AdminService.java] selectTopNotice()
    → AdminMapper 호출 후 ArrayList<AdminNotice> 반환

[QListController.java] selectQList()
    → aService.selectTopNotice() 호출
    → mv.addObject("noticeList", noticeList) 로 뷰에 전달

[qList/list.html]
    → th:each="notice : ${noticeList}" 로 테이블 행에 렌더링
```

**핵심 포인트**: SQL, Mapper, Service는 이미 구현 완료. 새로 만들 것 없음.  
`AdminService.selectTopNotice()` 를 그대로 재사용하면 됨.

## 4. 동작 설명

- 공지가 0건이면 공지바 자체가 렌더링되지 않음 (`th:if="${not #lists.isEmpty(noticeList)}"`)
- 기본 상태: 펼쳐진 상태 (공지 있을 때 바로 노출)
- 공지 클릭 → 질문 게시판의 공지 상세 페이지(`/qList/notice`) 로 이동 (기존 페이지 재사용)
- 공지 개수 뱃지는 Thymeleaf `#lists.size()`로 동적 계산
